THE DESIGN RULE (the "three jokes" pass, hardened over two telegraph tests):
  A good adventure-game reply lands THREE separate jokes:
    1. PREVIEW -- the short line in the menu. Funny on its own. A feint,
       an understatement, a register clash. (Joke you read.)
    2. SAYS    -- the longer line the player ACTUALLY delivers. It must add
       a NEW joke, not just say the preview louder: an escalation, a too-
       specific detail, a swerve into (embarrassed) honesty. (Joke you hear
       yourself say.)
    3. BOSS    -- the reply at the target node. It must TOP or SUBVERT the
       SAYS line, not just acknowledge it -- usually by latching onto the
       wrong word and synergising it. (Joke the other guy lands.)
  Two refinements learned the hard way:
    * NO PREVIEW-ECHO. The SAYS must not OPEN by restating the preview's
      words -- the preview is a label, the spoken line launches into a fresh
      beat. (Allowed exceptions: an interrupted preview the line completes-
      then-reverses, e.g. chair 2; or a single word re-stated to twist it,
      e.g. mindful 1's "Four dollars" -> "Four dollars. A year.")
    * KILL THE TELEGRAPHED. Each line was run past a cold "audience" that saw
      only the preview and tried to predict the payoff; anything it guessed
      (by mechanism, not wording) got rewritten until it surprised.
  If a node has several inbound options, its BOSS line is written as a
  generic deflection -- which is in character: Jordan deflects everything
  the same way. Single-inbound nodes get a BOSS line aimed at that one SAYS.

HOW TO READ / EDIT THIS:
  * Each [node] is one beat: the BOSS line, then the player's replies.
  * 'BOSS:' is the talking-head line (popup text, not voice-acted).
  * Each reply is:  PREVIEW (menu) / SAYS (spoken) -> the node it jumps to.
    If SAYS is omitted, the PREVIEW is spoken verbatim.
  * (ENDS CALL: stay) = you chickened out, call ends, nothing changes.
  * (ENDS CALL: quit) = the boss accepts -- this arms the leaving ending.
  * SPECIAL CASE: resign_final ends by cutting Jordan off mid-word. No final
    player button -- the call dies before anyone gets dignity. Leave it.
  * Edit the wording freely. Keep the [ids], the PREVIEW/SAYS labels, and
    the -> targets intact for a clean re-import; otherwise just scribble.
