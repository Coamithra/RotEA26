 SPRITE SCALE AUDIT  -  Revenge of the Evil Aliens (web port)

 Metric: EFFECTIVE SCALE at the 1440px render-target cap (presenter scale 2.4x).
 This is the WORST CASE the engine ever rasterizes (RenderScale.MaxHeight=1440,
 2.4 = 1440/600). Bigger windows just bilinear-stretch this target, so 2.4x is
 the true ceiling for how much a texture is ever magnified.

   eff = (on-screen size in 800x600 design px) * 2.4 / (native texels per frame)

   eff > 1  -> texture is MAGNIFIED at the cap -> soft -> UPSCALE candidate
   eff ~ 1  -> about right
   eff < 1  -> texture is MINIFIED -> wasted pixels -> SHRINK candidate
              (you can shrink the asset to ~eff of its current size, lossless@cap)

 NATIVE = per-frame texel width (sheet width / columns, or .dat source region,
 or full texture for single-frame). ONSCR = on-screen footprint width in design
 px (designFrameWidth * gameplay-scale, i.e. independent of window size).
 Dynamic-scale sprites use their MAX on-screen size (worst case for blur).

 Sorted by EFF, high -> low. Edit the STATUS column to track work.
 Generated 2026-06-22. (1080p window numbers = these * 1.8/2.4 = *0.75.)

 NOTES
 - "4096 cap" sprites (mediumship, mothership*): a single crisp@cap sheet exceeds
   the WebGL 4096px texture limit. Options: split the sheet, accept ~2x (eff ~1.2),
   or redesign. This is why UPSCALING.md left them native.
 - alienboss / spider boss are .dat multi-frame sheets: faithful HD needs N
   consistent frames + .dat regen (hard); the -premult repo has the originals.
 - brainlargetransglow is ONE texture shared by the boss (needs HD) and small
   braineroids (oversampled). Upscaling it helps the boss; small uses just get
   more oversampled (harmless). Net: upscale it.
 - "shrink" 4x sheets were built for the cap with headroom; dropping to ~3x stays
   crisp@cap and trims payload. Lower priority than the UPSCALE (eff>1) work.
