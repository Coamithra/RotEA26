# Evil Aliens preload manifest — read at level-load time by Compat/LoadProfiler.cs.
#
# Each line preloads one texture for one level, ON TOP OF that level's
# PreloadGraphicalContent() code list. It exists so newly-added art that isn't in
# the code list can be warmed without editing C#, and so the debug profiler can
# grow this file automatically (see below).
#
# Format:  <Level>|<asset-path>[|COLD|<ms>|<WxH>]      (# = comment, blanks ignored)
#   <Level>      a Levels enum name (Level1, Level2, TeamChallenge, ...)
#   <asset-path> Content-relative, lowercase, no extension (e.g. gfx/sprites/eye_idle)
#   trailing fields (optional) are profiler bookkeeping: COLD/WARM, worst decode ms,
#                  pixel size. The loader ignores them; only <Level>|<asset-path> matter.
#
# HOW TO GROW THIS FILE: run the game with ?loadlog, play the levels you care about
# (cold decodes are logged to the console as they happen and saved to localStorage),
# then run  eaPreloadExport()  in the browser console — it downloads an updated
# manifest. Replace this file with it and commit. Release builds only READ this
# file; they never record or append.

# Base-preload "dummy" component textures (deferred-load gap). GameScene.PreloadGraphicalContent
# constructs throwaway Explosion / BloodExplosion / Lazer / big-UFO instances at (-1000,-1000) to
# warm their art, BUT an AlienDrawableGameComponent loads its textures in LoadContent(), which XNA
# runs on the FIRST Update tick AFTER the component is added -- i.e. AFTER EndPreload() -- so those
# decodes land in live gameplay (logged COLD) instead of on the loading screen. The synchronous
# cm.Load() in ApplyManifest (inside the preload bracket) is what actually warms them. Scenes whose
# own PreloadGraphicalContent override already cm.Load()s these (Level3, InsaneBossI, Demo*, ...)
# are fine; SpaceDodge only preloads asteroids/ufos and TeamChallenge has no override at all, so
# both pay these cold mid-fight. Per object:
#   Explosion      -> block + explosionpurple   (explosion.cs LoadContent; explosionpurple 512x512 ~170ms)
#   BloodExplosion -> braingoo + blooddrop_green (the base dummy loads them even where no brain spawns)
#   Lazer          -> lazermiddle                (likewise, even where no JunkBoss spawns)
# Plus the UFO landed/stationary variants: UFO.Initialize ALWAYS cm.Load()s its stationarySprite
# (not just when it lands), so every spawned UFO decodes its landed sheet cold -- big UFOs
# (incl. the base dummy) use mediumship_landed (648x648 ~270ms), small UFOs randomly use
# ufometpootjes or smallship_landed. And the powerup HUD: ScoreVisualiser loads gfx/menu/powerbar,
# PowerupData loads the three HUD bars, all on first powerup -- a BonusSpawner runs in both scenes.
